# 🎯 核心理念

## 理念一：心智跃迁——从技术深度到心智广度

**核心原则**：
- **全球化成功的决定因素是心智广度**：技术能力是必要条件但非充分条件，心智跃迁才是决胜关键
- **从关注自我到理解他人**：从"我有什么好"转向"他们需要什么"
- **克服偏科和认知局限**：长期教育体系强调技术深度，但全球化需要更广泛的认知和人际能力

**心智跃迁维度**：

| 维度 | 技术思维（跃迁前） | 全球化思维（跃迁后） |
|------|------------------|-------------------|
| 关注焦点 | 我能做什么 | 他们需要什么 |
| 核心能力 | 算法精度 | 心智广度 |
| 沟通方式 | 展示产品优势 | 倾听客户需求 |
| 价值衡量 | 技术指标 | 信任与共鸣 |
| 竞争策略 | 速度和效率 | 理解和适配 |
| 视野范围 | 本地市场 | 全球生态 |

**跃迁路径**：
```
技术精通者 → 市场倾听者 → 文化理解者 → 信任构建者 → 全球领导者
```

**适用场景**：
- 企业出海战略规划
- 个人全球化能力评估
- 团队国际化转型

## 理念二：算法到受众——建立信任是核心

**核心原则**：
- **从Algorithm到Audience的转变**：全球化不是把技术推出去，而是让受众愿意接受
- **商业语言≠语言能力**：即使在社交场合建立信任和深入交流的能力，远超简单的语言翻译
- **信任建立是长期过程**：通过技术实力和透明运营逐步构建，而非表面公关

**信任建立四步法**：
```
1. 技术实力展示（入场券）：证明你能解决问题
2. 深入需求理解（建立连接）：听出他们要什么，而非只说你能做什么
3. 长期承诺表达（建立信心）：展示你会在当地长期存在
4. 文化共情实践（建立信任）：理解和尊重当地文化
```

**商业语言 vs 表面语言**：

| 层次 | 描述 | 示例 |
|------|------|------|
| 翻译层 | 能用外语表达 | 能介绍产品功能 |
| 交易层 | 能完成商业谈判 | 能谈价格和合同 |
| 社交层 | 能在社交场合建立人脉 | 能在party中自然交流 |
| 信任层 | 能让对方分享真实困难 | 对方愿意告诉你生意上的问题 |

**适用场景**：
- 跨文化商业沟通
- 海外客户关系建立
- 国际市场拓展

## 理念三：全球信任三层架构——数据透明、利益共赢、文化共情

**核心原则**：
- **信任是新的竞争壁垒**：当技术趋同时，信任成为决定性因素
- **三层架构缺一不可**：数据透明是基础，利益共赢是纽带，文化共情是升华
- **信任不可速成**：不是通过PR活动或聘请顾问建立，而是通过长期行动逐步构建

**三层架构详解**：

| 层次 | 核心内容 | 实践方式 | 验证标准 |
|------|----------|----------|----------|
| **数据透明** | 训练数据来源、用户数据处理政策、模型工作原理 | 公开数据处理政策、接受第三方审查、提供源代码托管 | 客户愿意将核心业务交给你 |
| **利益共赢** | 各方利益分配和合作模式 | 明确价值分配、给合作伙伴留合理利润、长期合作机制 | 合作方主动推荐你 |
| **文化共情** | 理解并适应当地文化和价值观 | 本地化团队建设、尊重当地习俗、讲述本地化故事 | 当地用户认同你的品牌 |

**信任保障技术措施**：
- 第三方代码托管（供应商跑路后业务仍可继续）
- 代码全面审查（确保无后门）
- 数据隐私保护（用户数据安全承诺）
- 长期存在承诺（在目标市场持续投入）

**适用场景**：
- 全球化信任策略
- 客户关系管理
- 数据合规与隐私保护

## 理念四：PMF新定义——用户相信你的使命

**核心原则**：
- **PMF是叙事而非简单的产品适配**：不是用户开始买你，而是用户相信你的使命
- **技术价值全球通用，情感价值本地化**：效率提升是全球需求，情感价值因地区而异
- **关注不变的需求**：效率和情感陪伴是未来10-20年不变的两大需求

**PMF新定义框架**：
```
传统PMF：产品适配市场 → 用户开始购买
新PMF：使命契合需求 → 用户相信你 → 持续购买并推荐

技术价值（全球通用）：
  - 效率提升 ✓
  - 成本降低 ✓
  - 功能丰富 ✓

情感价值（本地化）：
  - 美国 → 省时方便
  - 东南亚 → 社交连接
  - 中东 → 尊重与地位
  - 欧洲 → 隐私与自主
  - 拉美 → 娱乐与表达
```

**品牌使命一句话测试**：
- 能否用一句话概括"为什么做这件事"？
- 这句话是否在不同文化中都能引起共鸣？
- 用户听到这句话后，是否愿意相信你？

**适用场景**：
- 产品全球化规划
- 品牌定位
- 市场进入策略

## 理念五：不变需求法则——专注未来十年不变的东西

**核心原则**：
- **不要只关注每天变化的技术**：技术会过时，但人类核心需求不变
- **效率是永恒需求**：人类永远想用更少的时间做更多的事
- **情感陪伴是永恒需求**：技术越发达，人越需要真实的连接和陪伴

**不变需求 vs 变化技术**：

| 不变需求 | 技术实现方式（会变） | 本质 |
|----------|-------------------|------|
| 效率 | 从蒸汽机→电力→计算机→AI | 用更少资源做更多事 |
| 情感陪伴 | 从信件→电话→社交→AI陪伴 | 渴望被理解、被关注 |

**战略决策框架**：
```
面对任何新技术/新趋势时，问自己：
1. 它解决了什么不变的人类需求？（效率 or 情感）
2. 如果这项技术消失了，这个需求还在吗？
3. 10年后这个需求是否仍然存在？
4. 我的使命是否基于这个不变需求？

如果1-4都是"是"，那就是值得长期投入的方向
```

**避免的陷阱**：
- ❌ 追逐短期技术热点（"大家都在做"）
- ❌ 以为技术本身是产品（用户不在乎技术）
- ❌ 忽视不变需求，只看变化的技术
- ✅ 基于不变需求构建使命，技术是手段

**适用场景**：
- 长期战略规划
- 技术投资决策
- 品牌使命定义

## 理念六：从工程控制到领导信任

**核心原则**：
- **工程师思维是控制**：try-catch exception handling，保证所有情况都被覆盖
- **领导者思维是信任**：给予团队犯错空间，关注如何帮助本地文化和人民
- **从解决问题到激励解决问题**：领导者不亲自动手解决所有问题，而是激励团队去解决

**领导力转型框架**：

| 维度 | 工程师思维 | 领导者思维 |
|------|-----------|-----------|
| 核心模式 | 控制（Control） | 信任（Trust） |
| 面对未知 | try-catch覆盖所有情况 | 给团队犯错空间 |
| 能力焦点 | 个人技术能力 | 团队赋能和文化理解 |
| 问题处理 | 自己动手解决 | 激励团队解决 |
| 视野范围 | 技术愿景 | 如何帮助本地文化 |
| 竞争思维 | 你赢我输 | 合作共赢 |

**转型实践**：
1. **有意识地放手**：将任务交给团队，接受不完美
2. **关注人而非技术**：投入时间理解团队成员和当地文化
3. **提升愿景沟通能力**：让团队认同并追随，而非仅理解技术方案
4. **从局部到全局**：从关注单个技术问题到关注整个组织的全球化

**适用场景**：
- 技术人才转型管理
- 团队全球化领导
- 组织能力建设

## 理念七：本地化即全球化——出海是不断本地化

**核心原则**：
- **出海不是一个地区，是多个地区**：每个地区都有独特的文化、需求、付费动机
- **核心材料相同，形式因地区而异**：如同样的面粉在中国是煎饼、墨西哥是taco、法国是crepe
- **深入当地是唯一途径**：短期的考察团无效，需要在目标市场住3-6个月

**本地化深度阶梯**：

| 阶段 | 特征 | 时间投入 | 效果 |
|------|------|----------|------|
| 翻译层 | 界面语言翻译 | 1-2周 | 基础可用 |
| 功能层 | 功能适配本地需求 | 1-3月 | 市场进入 |
| 文化层 | 深入理解当地文化 | 3-6月 | 建立信任 |
| 情感层 | 讲述本地化情感故事 | 6月+ | 品牌共鸣 |

**市场进入建议**：
1. **住到当地去**：不是考察团，而是真正住在当地3-6个月
2. **与数百个当地人交流**：不是与几个商业伙伴，而是广泛了解当地文化
3. **一个市场一个市场做**：先做好一个，再扩展到下一个
4. **找到当地合作伙伴**：不是派工程师去试点，而是建立本地化团队

**付费动机地图**：

| 地区 | 核心付费动机 | 情感价值 | 产品叙事方向 |
|------|-------------|----------|-------------|
| 北美 | 省时方便 | 效率 | "节省你的时间" |
| 东南亚 | 社交连接 | 社交 | "连接你和你的朋友" |
| 中东 | 尊重与地位 | 尊重 | "彰显你的品味" |
| 欧洲 | 隐私与自主 | 自主 | "掌控你自己的数据" |
| 拉美 | 娱乐与表达 | 娱乐 | "释放你的创意" |

**适用场景**：
- 国际市场进入策略
- 产品本地化规划
- 跨文化产品设计

## 理念八：生态利他思维——不去赚最后一个铜板

**核心原则**：
- **生态成功是利他选择**：给合作伙伴留有合理利润，才能构建可持续生态
- **不赚最后一个铜板**：短期利益让步换来长期合作关系和生态繁荣
- **战场、时机、方式需战略选择**：每家企业出海方式不同，需量身定制

**利他思维决策框架**：

| 决策维度 | 短期思维 | 利他思维 |
|----------|----------|----------|
| 利润分配 | 利润最大化 | 合理分配，留空间给伙伴 |
| 合作关系 | 利用合作伙伴 | 赋能合作伙伴 |
| 市场策略 | 快速占领 | 稳健深耕 |
| 技术分享 | 封闭垄断 | 开放共享 |
| 竞争态度 | 零和博弈 | 共生共赢 |

**战略三选择**：
1. **战场选择**：先进入哪个市场？为什么？
2. **时机选择**：什么时候进入？市场是否成熟？
3. **方式选择**：独立运营还是合作？本地团队还是派驻？

**适用场景**：
- 生态战略规划
- 合作伙伴管理
- 全球化路线设计

## 理念九：身体是全球化创业的地基

**核心原则**：
- **创业是十年二十年的长跑**：不是百米冲刺，而是马拉松
- **身体是1，其他都是0**：没有身体健康，一切战略和能力归零
- **定期锻炼不可忽视**：再忙也要抽出时间维护身体

**长期创业者健康框架**：
```
身体健康（地基）
  → 持续工作能力
    → 心智稳定
      → 决策质量
        → 全球化战略执行
          → 长期成功

如果地基不稳，上层全部崩塌
```

**实践建议**：
- 每周至少3次运动，每次30分钟以上
- 保证基本睡眠（6-8小时）
- 定期体检，关注长期健康指标
- 在高压时期更加注意身体信号

**适用场景**：
- 创业者自我管理
- 长期战略执行
- 团队健康文化

# 💡 设计心法

## 心法一：速度不是唯一的关键

**核心**：如果速度是唯一关键，那么最成功的企业应该在成本最低的地方，而非在最有共鸣的地方。

**应用方法**：
1. **质量 vs 速度**：在全球化中，质量（信任、共鸣、文化适配）比速度更重要
2. **共鸣 vs 规模**：用户认同你的使命比用户数量更有价值
3. **长期 vs 短期**：十年之功的投入比短期冲刺更有回报

---

## 心法二：到当地去住，不要只参观

**核心**：真正的文化理解需要深度沉浸，短期的考察团只能看到表面。

**应用方法**：
1. **至少住3-6个月**：在目标市场真正生活
2. **与数百个当地人交流**：不是与几个商业伙伴，而是广泛了解
3. **像当地人一样生活**：体验当地的生活方式和文化

---

## 心法三：核心不变，形式万变

**核心**：全球化产品有不变的核心（效率/情感），但形式因地区而异。

**应用方法**：
1. **识别不变的核心**：你的产品本质解决什么不变需求？
2. **适配本地形式**：同样的核心价值，用当地文化和习惯来表达
3. **避免一刀切**：不同地区需要不同的产品叙事和交互方式

---

## 心法四：AI的全球化不以任何人的意志为转移

**核心**：AI从诞生之初就是全球的，地缘政治无法阻止技术全球化的本质趋势。

**应用方法**：
1. **以全球视野构建产品**：不要从"中国市场出海"的视角，而是从"全球产品"的视角
2. **寻找全球共鸣**：使命和价值观应超越国家边界
3. **拥抱开放生态**：通过开源和共享建立全球合作

# 行业深度对话分析框架

## 概述

本技能提供系统化的方法论，用于分析任何行业的深度对话内容（圆桌会议、专家访谈、行业辩论等），并提取可操作的市场洞察和趋势预测。

核心理念：从多角度对话中识别**共识**、**分歧**和**盲点**，构建行业发展的立体画像。

## 核心分析框架

### 第一步：识别参与视角

任何深度行业对话都包含多个专业视角。常见视角类型：

- **投资视角**：资本逻辑、估值逻辑、投资回报、风险判断
- **技术视角**：技术路线、研发重点、技术瓶颈、创新方向
- **产品/应用视角**：场景落地、用户需求、产品迭代、商业化路径
- **市场/商业视角**：市场规模、竞争格局、商业模式、盈利路径
- **政策/监管视角**：政策影响、监管要求、合规成本、政策红利
- **供应链/产业视角**：产业链结构、供应瓶颈、成本结构、产能布局

**操作指南**：阅读对话时，先标注每位发言者的核心视角，构建"视角-观点"矩阵。

### 第二步：提取核心议题

从对话中识别反复出现、引发争议或达成共识的核心议题。使用以下模板记录：

```markdown
### 议题：[议题名称]

**争议点**：
- 观点A：[描述] (支持者：...)
- 观点B：[描述] (支持者：...)

**共识**：
- [达成的共识1]
- [达成的共识2]

**未知/盲点**：
- [尚未厘清的问题]
```

### 第三步：识别核心资源与瓶颈

每个行业都有其关键资源约束。从对话中识别：

1. **数据资源**：数据获取、数据质量、数据成本
2. **算力资源**：计算能力、算力成本、算力分配
3. **人才资源**：关键技能人才、人才培养周期
4. **资本资源**：融资环境、投资热度、资本回报周期
5. **供应链资源**：核心零部件、产能、供应商议价能力
6. **政策资源**：牌照、资质、政府支持

**分析方法**：对于每种资源，记录：
- 当前供给状况
- 未来需求预测
- 成本结构变化趋势
- 瓶颈突破路径

### 第四步：竞争格局分析

从对话中提炼竞争格局的关键维度：

**市场参与者分类**：
- 巨头/大厂：战略意图、资源优势、劣势
- 创业公司：聚焦领域、差异化、生存策略
- 垂直玩家：细分领域、护城河、成长空间

**竞争维度**：
- 技术路线竞争
- 场景落地竞争
- 人才争夺
- 资本博弈
- 供应链整合

**生存法则**：
- 创业公司如何与巨头共存？
- 垂直整合 vs 生态合作？
- 先发优势 vs 后发制人？

### 第五步：未来趋势预测

基于对话中的观点，构建未来12-24个月的预测框架：

**预测维度**：
1. **技术突破**：哪些技术会在什么时间窗口成熟？
2. **商业化落地**：哪些场景会率先实现商业闭环？
3. **市场格局**：市场份额会如何变化？淘汰赛何时开始？
4. **成本下降曲线**：核心成本项会如何下降？
5. **监管政策**：政策会如何演变？

**预测模板**：
```markdown
### 预测：[预测内容]

**时间窗口**：[12个月/24个月]
**关键前提**：[实现该预测的必要条件]
**验证指标**：[如何验证预测是否成立]
**影响分析**：[对行业格局的影响]
```

# SuperLeads 企业线索分析器

基于项目线索库（SuperLeads）知识库的智能分析技能，提供企业信息检索、商业模式分析、投资标的筛选等专业服务。

## 核心能力

### 1. 企业画像分析
- 企业基本信息查询（名称、规模、业务范围）
- 核心技术梳理
- 团队背景分析
- 发展历程追踪

### 2. 竞品对标分析
- 同行业企业对比
- 技术路线差异分析
- 商业模式对比
- 市场定位分析

### 3. 投资标的筛选
- 行业赛道筛选
- 融资阶段匹配
- 技术门槛评估
- 市场潜力判断

### 4. 销售线索构建
- 目标客户画像
- 决策链路分析
- 合作切入点识别
- 谈判策略建议

## 使用方法

调用 ima skill 的 `search_knowledge` 接口搜索知识库内容：

```bash
# 搜索企业信息
知识库ID: -70tg7E8b0rjztuOEX9RDdg2kjGMdr0nVR0ofH5qa2M=
搜索接口: openapi/wiki/v1/search_knowledge
```

### 典型应用场景

**场景1：企业基本信息查询**
- 用户："帮我分析一下做AI新药开发的公司有哪些？"
- 执行：搜索关键词 "AI 新药开发"，整理企业列表和关键信息

**场景2：竞品对比分析**
- 用户："对比一下国内外创业方向差异"
- 执行：搜索关键词 "创业方向"、"国内外对比"，生成对比报告

**场景3：投资标的研究**
- 用户："推荐一些AI硬件领域的投资标的"
- 执行：搜索 "AI硬件"、"机器人"、"芯片" 等关键词，筛选符合投资标准的企业

**场景4：销售线索筛选**
- 用户："帮我找一些做3D打印的潜在客户"
- 执行：搜索 "3D打印"，提取企业联系方式、决策人、业务规模等信息

|------|------|

| 核心技术 | 技术栈、专利、技术壁垒 |
